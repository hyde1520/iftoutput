Comparing Landscapes to Develop Treatment Alternatives

Developing Treatment Alternatives facilitates evaluation of fuel treatment alternatives and their spatial arrangement on the landscape. It allows you to create several edited versions of a landscape or treatment area, run a fire behavior simulation for some or all of the edited landscapes, and compare the results for each treatment on fire behavior spatially, and create reports with charts and tables if desired. The task is divided into five steps:

  • Pick a landscape
  • Edit the landscape
  • Enter model inputs
  • Run the model for some or all of the edited landscapes
  • Compare model outputs for your treatment alternatives

All edited landscapes, resultant model run outputs, and comparison reports created in this workflow will be filed in your workspace in the same location as landscape you choose in the following step as your originating landscape.

To view both your landscape, and editing rules at the same time, use the View Split Map button at the top left of the page.

Throughout these steps we refer to different landscape (Originating, Existing, etc.). If you need a quick reference to remember what each of these refers to, see the LAbout Landscapes in IFTDSS Topic.

Choose an Originating Landscape

Use the Pick a Landscape tab to create or select a starting unedited landscape. The landscape selected in this step initiates your comparison session and will be marked with “0” throughout the workflow. The “0” indicates the original landscape extracted from LANDFIRE that has not been edited. In subsequent tabs, all landscapes that match this extent will be available for you to work with. In the next tab, the Edit Landscape tab, you may edit any of these landscapes.

Landscape “0” defines the landscape extent through the rest of the comparison process. You may also use it to represent your existing landscape if desired, or edit it in the next step (editing) to represent your Existing Condition (EC). You may also work from any other available files that match the extent of landscape "0".

To complete this step either:

Use the Select Landscape drop-down menu to select an existing un-edited landscape from your workspace or,

Use the Create on Map option to draw a new landscape extent and extract a new Landfire landscape

Edit Your Landscape

Here you will edit landscapes to develop your treatment alternatives. The landscape(s) available here will include:

  • your unedited LANDFIRE landscape selected in the previous step (these will be referred to as the “0” landscape)
  • and any landscapes matching that extent.

Create your treatment alternatives by altering your your Existing Condition landscape.

You may use Default Treatment/Disturbance Rules to edit and create your alternatives, or custom rules to create your alternatives.

Each option is described in detail at the Editing a Landscape to Develop Treatment Alternatives Topic.

It is important that the landscape you use to develop and compare treatment alternatives mimics, as closely as possible, conditions as they exist on the ground. This ensures more accurate comparison results in gauging how each alternative does or does not meet objectives. The EC may need to reflect new disturbances on your landscape, like recent wildfires, fuels treatments, or windfalls.

If you had not created an Existing Landscape yet, that could also be done here, and then edited to represent treatments.

You may create your own landscape edit rules by selecting specific conditions, and specifying what modifications to make where ever those conditions occur.

To split the view and display your mapped area on the left, and this interface on the right, select the show split map button at the top left of the page.

  1. Begin by selecting your Starting Landscape which you would like to edit
  2. View the Add Landscape Rule options by either selecting the Add Landscape Edit Rule + sign just below the starting landscape field, or select the dropdown arrow to the far right of the Add Landscape Edit Rule heading.
  3. Specify the conditions you would like to select in the fields below Where these conditions exist. In this example we have selected 'Fuel Model' as the Attribute, 'is equal to' as the Operator and fire behavior fuel model '165 (TU5)' as the Value. This selected all TU5 fuel modeled areas across the landscape.
  4. Specify what you would like change conditions to by populating the fields below Modify the following values. In this example we have selected 'Canopy Cover' as the Attribute, 'decrease by' as the Operator, and '5' (for 5 percent) as the Value. For all TU5 fuel modeled areas, we will decrease the canopy cover by 5 percent.
  5. The Apply to Landscape Mask (optional) dropdown allows you to select a shapefile to which these edits will be contained. If this is left blank, the edits will be applied to the entire landscape.
  6. When you are ready to apply this rule select the Add to Rules button. Notice the rule has been added to you list, and the order noted. You can delete rules before saving by using the delete button to the right of each rule.

    7. Rules are applied top to bottom, first by Default Fuels Treatment / Disturbance Edit Rules then by user defined Landscape Edit Rules. Currently there is no way to order your rules other than by the order in which you enter them, so be deliberate about how you apply rules and in what order you enter them.

    As rules are added, will see them listed at the to of the page, along with the order they have been added, and a notification to let you know the new rule has been successfully saved.

  7. Once you have all the rules, in the order you would like them applied, give your landscape a descriptive name and select Save New Landscape. Your original landscape will remain, and an edited copy will now be available throughout IFTDSS.

Continue this process to create as many landscapes as needed to compare. When you are finished editing and creating new landscapes, continue on to the Model Input tab to establish conditions under which the landscapes you just created will be modeled.

Establish Model Inputs

Use this tab to establish the model state by which all landscapes in your comparison session will be run and compared. If you are unsure as to what your model state should be for your comparison session, utilize My Modeling Playground to test different weather and fuel moisture conditions on your landscape(s). Creating an Auto97th Landscape Fire Behavior report in the Landscape Summary task of the Landscape Evaluation stage of the planning cycle. can also help give you a starting point for establishing model inputs.

Once a model is run in Landscape Comparison, it is permanent, there is no way to re-run that model. For this reason, testing out various modeling inputs in My Modeling Playground before proceeding with the rest of Landscape Comparison is recommended.

To complete this step:

  1. Type wind speed and wind direction in the Wind section
  2. Select Crown Fire Method and enter Foliar Moisture Content in the Crown Fire Inputs section
  3. Populate the timelag and herbaceous and woody fuel moisture percentages in the Initial Fuel Moisture section (Optional)
  4. Add rows to establish fuel moistures for individual fuel models
  5. Save Inputs

For full details about the fire behavior modeling, inputs, and outputs, visit the Fire Behavior Basic Topic.

You can adjust model inputs even after they have been saved during your session as long as you have not run a model.

Once you have run a model in the Run Model tab with saved inputs, they cannot be modified in this session.

If you are unsure what you should use for model inputs, utilize the IFTDSS Playground to test out different weather and fuel moisture condition scenarios and/or the Auto97th Landscape Fire Behavior run and reports in the Landscape Evaluation>Landscape Summary portion of the Cycle.

Run Your Model

The Run Model tab provides a space where you can re-name and run models for landscapes you created in the Edit Landscape tab. The Landscape column in the Run Model table will list landscape “0” as selected in the Pick Landscape tab and all of its derivatives. Again, as mentioned in previous tabs, landscapes edited off of landscape “0” outside of this workflow will be available here as well. You can model and compare those landscapes just the same as if you created them within this session. The Run Name box will be auto populated with the landscape name and the type of model being run.

Currently IFTDSS only runs Landscape Fire Behavior but more models will be integrated soon.

Feel free to accept the default name or rename your model run. Now you can produce Landscape Fire Behavior for landscapes of your choosing by selecting Run Model after the landscapes you would like to compare. Use the refresh button to check status.

To complete this step:

  1. Click the Refresh button next to Landscape in the Run Models table to be sure all of your landscapes are listed
  2. Click Run Model for landscapes you would like to compare
  3. Click the Refresh button next to Run Status in the Run Models table to check the status of your model outputs
Compare Alternatives

The Compare Alternatives tab lists all the landscapes we have included in the comparison process. Note landscapes that have had fire behavior runs modeled for them are indicated with green checkmarks.

To compare these landscapes, use the checkboxes next to the landscapes you wish to compare:

  • Select the checkbox for two landscapes and model runs you would like to compare
  • The first box you check will be tagged with “1” and should represent your “treated” landscape.
  • The second box you check will be tagged with “2” and should represent the initial landscape you are comparing against, such as the untreated existing condition landscape.

Example: My objective is to reduce flame length on my treated landscape. So, I need to show change due to treatment. I need my output grid to show that flame length was reduced by treatment. So, AltA(FL 2)-EC(FL 10)= FL -8. I reduced my FL by 8. Grid will show -8 meaning an 8 foot reduction.

Once you have checked 2 boxes you can:

  • View on Map where, if it is not already open, will open the split screen and will display the difference grid for the landscapes you choose. The layers list should also be open and will list the two landscapes, two model outputs, and two difference grids (landscape diff and model output diff) to view.
  • Compare in Summary Report where a pop up box will come up for you to select Create Report for either or both of Landscape Compare or Fire Behavior Compare. Use the refresh button to check status. Once the report has been created you can click on the blue hyper link opening a new tab where you can view, download or print your report.
